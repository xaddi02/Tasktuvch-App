¿Qué es la POO?

La Programación Orientada a Objetos (POO) es un paradigma de programación que utiliza "objetos" para diseñar aplicaciones. Se basa en 4 pilares fundamentales:

  • Clases: Son las "plantillas" o moldes. Definen las características (atributos) y acciones (métodos) que tendrán los objetos.
  • Objetos: Son instancias (creaciones reales) de una clase.
  • Encapsulamiento: Oculta los datos internos de un objeto para protegerlos, permitiendo modificarlos solo mediante métodos seguros (getters/setters).
  • Herencia: Permite crear nuevas clases basadas en clases existentes, reutilizando y extendiendo su código.
  • Polimorfismo: Permite que un mismo método funcione de diferentes maneras dependiendo del objeto que lo invoque.

Ejemplos de Uso

Imagina que estás modelando datos de la vida real. Una clase es la plantilla Perro, y un objeto puede ser Miki.

class Perro {
    String nombre;
    String raza;

    // Método constructor
    public Perro(String nombre, String raza) {
        this.nombre = nombre;
        this.raza = raza;
    }

    public void ladrar() {
        System.out.println("¡Guau! Soy " + nombre);
    }
}

// Instanciando un Objeto usando 'new'
Perro miMascota = new Perro("Miki", "Husky");
miMascota.ladrar();
